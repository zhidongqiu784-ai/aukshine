# D:\wayne 项目规则

## NocoBase 项目开工前必须连接当前环境

处理 NocoBase 项目任务时，不能只根据记忆、文件名或用户简述猜测当前项目结构。动手前必须连接当前 NocoBase 环境，了解本次任务涉及的真实数据模型和业务配置。

## NocoBase 使用 MySQL 数据库

当前 NocoBase 环境使用的是 MySQL 数据库。涉及数据查询、SQL、字段类型、表结构、统计口径、公式计算、数据写回、工作流 SQL 节点或接口取数时，必须按 MySQL 的真实表结构和语法确认后再处理。

### MySQL 相关要求
- 不得按 PostgreSQL、SQLite 或其他数据库的字段类型、函数、引号规则、分页语法或 JSON 语法进行猜测。
- 编写或修改 SQL 前，必须确认真实表名、字段名、字段类型、索引、关联关系和样例数据。
- 涉及日期、时间、JSON、聚合、分组、排序、分页、空值判断时，必须使用 MySQL 兼容写法。
- 涉及写回、更新、删除、批量改数据时，必须先按“修改前确认规则”和“碰生产环境前先停”执行。

### 必须先确认的信息
- 本次任务涉及哪些 collection、字段、关系字段和关联表
- 字段真实名称、字段类型、展示名称、是否必填、默认值和枚举值
- 相关页面、区块、操作、筛选条件、联动规则和权限配置
- 相关工作流、触发条件、节点配置和写回逻辑
- 必要时查看少量样例数据，确认数据形态、空值、重复值和边界情况

### 禁止行为
- 不得凭猜测编写字段名、表名、关系名、API 路径或工作流节点
- 不得在未确认 schema 的情况下修改数据写回、公式计算、字段映射或页面配置
- 不得把旧项目经验直接套用到当前 NocoBase 环境
- 不得把无法连接环境时的推测说成已经确认

### 无法连接时的处理
如果当前无法连接 NocoBase 环境，必须停止实施修改，并明确说明：
- 哪些关键信息无法确认
- 继续修改可能带来的风险
- 需要用户提供什么连接方式、权限、页面路径或样例数据

## 备份文件存放规则

- 创建文件备份时，必须在被备份文件所在目录下创建或使用 `备份` 子文件夹，并将备份文件存放在其中。
- 禁止将备份文件与正式文件混放在同一层目录。
