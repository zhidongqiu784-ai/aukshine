## 业务目的

该工作流每天根据历史合计销量校准基础活动类型系数，再把最新系数同步到今天及未来的水位表记录，最后重算预计销量。

## 执行流程

### 1. 自动计算活动系数

读取今天以前 `daily_sales` 中 `shop = '合计'` 的历史记录，按 ASIN、国家和单一日类型分组计算：

`活动系数 = SUM(sales) / SUM(weighted_sales)`

结果保留两位小数并写入 `sales_coefficient.coefficient`。该步骤只计算配置分类为“基础活动类型”的日类型，并排除复合日类型、空销量、空加权销量和加权销量为零的记录；“日常”系数不更新。

### 2. 同步今天及未来的日类型系数

读取今天及未来 `daily_sales` 中实际出现的日类型，统一中文顿号、中文逗号、英文逗号及空格，并把 `BD（预测）`、`BD(预测)` 归一为 `BD`。

复合日类型会拆成多个组成部分，分别从 `sales_coefficient` 取值后连乘，结果保留两位小数。只有全部组成部分都有唯一有效系数时才更新 `daily_sales.coefficient`，且只写入值发生变化的记录。

### 3. 更新预计销量

对今天及未来的全部 `daily_sales` 记录执行：

`maybe_sales = weighted_sales * coefficient`
