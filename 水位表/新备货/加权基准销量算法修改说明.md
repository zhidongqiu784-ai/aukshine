## 一、修改原因

现行机制 = 前 30 天分段渐进权重算基准,活动预估 = 基准 × 上一轮对标翻倍系数。两个系统性失真:

1. **销量水平位移后基准反应太慢**:B0GMD43632 在 2026-07-07 后实际日销掉到 1~9 件,系统连续 10 天给 13~25 件,高估 2~4 倍。分段权重对台阶式掉档/跳档只能缓慢稀释,高估直接污染下单和发货。
2. **活动翻倍系数被单轮异常污染**:只对标上一轮,若上一轮起点基准恰好虚低(断货后/淡季刚回),翻倍被高估,下一轮预估爆表。实测同一 ASIN 的 BD 实际翻倍在 0.6~7.0 之间波动,单轮对标不可靠。

---

## 二、时序铁律(先读这条再看算法)

- **第 D 日预估只使用截至 D-1 日收盘的数据。** D 日实际收盘后才进入基准/系数状态,供 D+1 日使用。
- 基准更新、重锚判定、活动轮关闭,只发生在"日期已关账"之后;**当天未跑完的销量一律不参与**(按站点销售日定义关账,中国时间上午跑任务时美国当天还没结束)。
- 盘中(每小时)工作流只做数据同步和当日预估值的展示,**不得**用盘中销量更新基准、触发重锚或结算活动轮。

## 三、实施形态(强约束)

**每日关账后确定性全量重放**:每天每 ASIN 从数据起点(或固定回看窗口,建议全历史,562 天量级的纯算术开销可忽略)按日期顺序重算一遍,结果按「ASIN+国家+日期」唯一键覆盖写入。
- 好处:天然幂等,重试/并发/补录历史数据都不会造成状态漂移,不需要维护增量状态的事务锁。
- 明确**不采用**"每小时增量更新状态"方案:重锚候选、冷却期、活动轮状态在增量模式下会被同一天的多次执行重复消耗(单日 LD 上午跑一次就提前结轮)。
- 输入序列先规整:每「ASIN+国家+合计行+自然日」唯一一行(重复日期取最后一条并告警);**所有"天"都指自然日**,衰减按实际日期差计算,"连续"必须日期+1。

## 四、数据有效性规则

| 情况 | 处理 |
|---|---|
| 有效日 | 销量非空 且 非断货日 且 系数合法。只有有效日进基准、进重锚候选、进活动轮均值 |
| 断货日(out_of_stock=1) | 不进基准、不进重锚候选、不进轮均值;不计入误差统计 |
| 销量为空(未同步) | 视为"未知",跳过,不当 0 处理;打断重锚候选的"连续" |
| 季节系数为空 | 按 1 处理 |
| 季节系数 ≤0 / 非数字 | 该日无效,计数告警,**禁止静默当 1** |
| 缺天(日期不连续) | 视为未知天:打断重锚候选连续性、打断活动轮连续性;衰减按真实日期差算 |
| 负销量 / 非数字 | 入口校验拒绝,告警 |
| 0 与空值判断 | 一律显式判"是否为空",禁止用真假值判断把 0 当空 |

## 五、算法定义(三件套)

### A. 基准销量:指数衰减 + 截尾 + 位移重锚

1. **折算值** = 当日销量 ÷ 当日系数(非活动日 ÷ 季节系数;活动日 ÷ 该轮冻结的活动系数,见 B)
2. **基准(截至某日)** = 近 30 个自然日内有效折算值的指数衰减加权平均:权重 = 0.95^(相隔自然日数) × 日权重;活动日日权重 0.5,非活动日 1.0
3. **单日截尾**:折算值入库前夹在 [前日收盘基准×0.5, 前日收盘基准×1.5]
4. **位移重锚(冻结参照状态机,治"截尾+慢衰减跟不上台阶变化")**:
   - 某有效非活动日的**未截尾**折算值偏离当时基准超 30% → 开候选,**冻结该日基准作参照**(后续都和这个冻结值比,不和滑动基准比——否则基准边跟边比,名义 30% 实际要 50%+ 才触发,阈值死区)
   - 之后每天:日期连续(+1)、同向、对冻结参照偏离超 30% → 计数 +1;断向 / 缺天 / 活动日 / 断货日 / 无效日 → 候选清零
   - 计满 5 天 → **重锚**:历史清空,以这 5 天未截尾折算值重建基准(同一衰减规则加权,最近一天权重最高);**截尾参照同步重置为新基准**(否则重锚当天还被旧基准×0.5 托底,慢一天收敛);冷却 = 重锚日 + 5 自然日内不开新候选(存"冷却截止日期",不用计数器)
   - 状态量:候选起点日期、冻结参照值、候选方向、已计数天数、冷却截止日期、最近重锚日期。全量重放模式下这些只是重放过程的中间变量,天然可复现
5. **断货保护**:断货日不进候选,连续断货零销**不会**触发向下重锚(否则断货 5 天会被误判为需求暴跌直接锚到 0)

### B. 活动系数:轮次状态机 + 近 3 轮中位数 + 夹紧

1. **轮的定义**:同活动类型(BD / LD / PD)且**日期连续**的一段;缺天或换类型即断轮。单日 LD 自成一轮。类型识别用日类型字段的受控枚举显式映射(禁止靠子串顺序碰运气;复合类型明确主类型)
2. **开轮冻结三元组**,整轮不变:
   - 轮起基准 = 开轮前一日收盘基准
   - 轮系数 =(见下)
   - 轮初预估 = 轮起基准 × 轮系数
3. **轮系数**:取该 ASIN 该类型**已关闭**轮次的实际翻倍——0 轮:类型默认系数;1 轮:取其值;2 轮:算术平均;≥3 轮:近 3 轮中位数(排序取中间值)。结果夹紧到 [类型默认×0.5, 类型默认×3]
4. **实际翻倍** = 该轮有效日日均实际 ÷ 轮起基准;轮起基准为空或 ≤0 → 该轮不记翻倍。分母用 A 的重锚基准,起点不虚低,翻倍就不虚高——治第 2 个失真的根子
5. **只有轮结束后才记系数**(次日关账确认类型改变/缺天)。进行中的轮绝不写入,每轮只写一次(全量重放天然满足;增量实现会在盘中反复结轮,这是禁用增量的原因之一)
6. 类型默认系数读现有销售系数配置表;**记录取数时生效的配置版本**,历史重放用当时值(至少:配置变更留痕,验收期内不改配置)

### C. 轮内动态修正

活动第 k+1 天(k≥1)预估 = (2 × 轮初预估 + k × 前 k 天轮内日均实际) ÷ (2+k);首日 = 轮初预估。只用**已关账完成**的活动日实际。

### 冷启动阶梯

- 0 个有效日:无基准,预估为空(前端显示可用销售人工值或旧算法兜底,由 IT 定;v3 字段留空不硬造)
- 1~29 个有效日:正常按上述公式算(数据少时基准自然偏向近几天),重锚照常允许(新品爬坡快,重锚正是跟涨手段)
- 首轮活动若开轮时无基准:轮初预估为空,该轮不记翻倍

### 参数表(调参样本上调优,留出样本验证过不劣化)

| 参数 | 值 |
|---|---|
| 衰减 | 0.95^自然日 |
| 窗口 | 30 自然日 |
| 截尾带 | 前日基准 × [0.5, 1.5] |
| 活动日权重 | 0.5 |
| 重锚 | 连续 5 自然日、对冻结参照同向偏离 >30%、冷却 5 自然日 |
| 轮系数 | 近 3 轮(1轮取值 / 2轮均值 / 3轮中位数),夹紧 [默认×0.5, 默认×3] |
| 回测所用类型默认 | BD 1.75 / LD 2.5 / PD 3.6(实现时读配置表) |

- v3_weighted_sales(v3基准销量,数值,2位小数)
- v3_maybe_sales(v3预估销量,数值,1位小数)

字段名/类型/精度可按现有 V2 系列字段惯例调整,但新旧算法必须并行可对比。

**验收三步:**
1. **快照复现**:附件 snapshot_*.csv 是 5 款 ASIN 的冻结输入(拉取日 2026-07-21)。实现完成后用**同一份快照**喂新实现,逐日输出应与 backtest_v2_*.txt 的"新算法v2"列一致(容差 ±0.1,纯算术无随机)。不要用线上实时数据验收——数据会增长、会补录,对不上说明不了问题。
2. **影子并行 2~4 周**:v3 字段与现行字段同时跑,全量 ASIN,按周对比平均绝对误差与偏差(活动/非活动分开);v3 不劣于现行再谈切换。
3. **切换另立决策**:并行数据出来后由 Ailah 拍板,本文不含切换步骤。

**精度约定**:中间计算保持浮点,只在最终写入时舍入(四舍五入);禁止逐步舍入累积误差。
