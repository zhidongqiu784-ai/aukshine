# 工作流：发货计划演变-v2-重算现有建议

## 业务目的

对已经存在的 W1-W7 模拟计划重新计算建议数量，再把新建议同步到 V2 库存与在途推演。

本目录中的正式 SQL 与线上工作流节点一一对应。线上工作流只有两个 SQL 节点，不需要单独的预览节点。

## 节点顺序

1. `01-重算现有W1-W7建议数量.sql`：原地更新现有 W1-W7 建议数量，不新增、不删除模拟计划。
2. `02-重算V2库存与在途.sql`：建议量更新后，全量重算 `daily_sales.v2_*` 水位数据。

执行顺序固定为节点 01 -> 节点 02，不能只执行其中一个节点。

## 保护条件

- 只处理 `plan_source = 'shipment_plan_v2'`、`shop = '合计'`、发货日不早于当天的计划。
- 任一条计划已关联真实发货时，整组跳过。
- 任一条计划存在活动申请时，整组跳过。
