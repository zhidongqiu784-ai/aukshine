# 推新板块功能公式总结

## 1. 板块一句话说明

推新板块用于跟踪新品或重点 ASIN 在每天的推新阶段表现：当前价格、活动、实际单量、广告单、自然单、目标转化率、预计测评量、实际测评量、关键词词位和复盘计划。

这个板块的主线是“每天应该测多少、实际测了多少、实际转化率是否达标、关键词计划是否执行”。

## 2. 主要功能

### 2.1 基础展示

一行代表一个 `国家 + ASIN + 日期`。

基础数据来自：

- `daily_asins`
- `weekly_performance`
- `daily_keyword_tracking`
- `new_eval_words`
- `new_eval_words_daily`

### 2.2 关键词管理

推新板块会根据 `new_eval_words` 动态生成关键词列。

每个关键词下有 6 个子字段：

- 预计刷单
- 预测词位
- 实际刷单
- 实际词位
- 复盘
- 新计划

### 2.3 测评计划判断

推新板块会自动计算：

- 预计刷单总数
- 单日测评应需量
- 实际自然单
- 计划测评缺口或超量
- 实际测评缺口或超量
- 实际 CVR 是否达标

### 2.4 周汇总

推新板块会把同一自然周的日数据生成周汇总行，写入 `daily_weekly_summary`。

周汇总范围：

- 周起始：周日
- 周结束：周六
- 日期筛选会扩展到完整自然周

## 3. 数据什么时候抓

### 3.1 页面打开时即时抓取

页面加载、筛选变化、翻页、刷新时会读取：

- `daily_asins:list`
- `weekly_performance:list`
- `daily_keyword_tracking:list`
- `new_eval_words:list`
- `new_eval_words_daily:list`
- `daily_weekly_summary:list`

同时会用测评需求、RSG、返款、订单表同步关键词实际刷单：

- `evaluation_requirement:list`
- `rsg_daily:list`
- `refund:list`
- `order_list:list`

### 3.2 自动任务抓取时间

脚本中记录的自动来源如下：

- `country`、`asin`、`date`、`model`、`sale_owner`：每日 8:30 从 ASIN 表生成，只包含状态为「新品、重点、普通」的产品。
- `date`：每日 8:30 自动生成从当天起未来 3 个月的日期。
- `activity_annotation`：每日 8:30 自动匹配 BD/LD 活动日期。
- 非 US/CA 站点的购物车价、LP/WP/TP、售卖账号、推广天数：每日 8:30 自动同步昨日数据。
- US/CA 站点的购物车价、LP/WP/TP、售卖账号、推广天数：每日 13:00 自动同步昨日数据。

### 3.3 默认抓取时间范围

页面默认日期筛选为「近7天及以后」。

字段名公式版：

- `start_date = today - 6 days`
- `end_date = null`
- `daily_asins.date >= start_date`
- 页面读取时会执行 `expandDateRangeToNaturalWeeks(getDateRange)`

中文公式版：

- 默认从页面打开当天往前 6 天开始，一直取到未来已有数据。
- 实际抓取时会扩展到完整自然周，保证周汇总能完整计算。

### 3.4 可选日期范围

- 全部日期：不加日期条件。
- 今天：`date = 今天`
- 昨天：`date = 昨天`
- 近 7 天：`今天 - 6 天 <= date <= 今天`
- 近 30 天：`今天 - 29 天 <= date <= 今天`
- 近 90 天：`今天 - 89 天 <= date <= 今天`
- 本月：`本月 1 日 <= date <= 今天`
- 上月：`上月 1 日 <= date <= 上月最后一天`
- 自定义：用户选择日期后，再扩展到自然周。

## 4. 数据覆盖与写回逻辑

### 4.1 公式写回表

推新板块的核心公式写回 `daily_keyword_tracking`。

写回字段：

- `est_review_total`
- `daily_eval_demand`
- `actual_natural_order`
- `plan_eval_judgment`
- `actual_eval_judgment`
- `actual_cvr_judgment`

### 4.2 订单链接联动写回

推新板块还会联动写回 `daily_order_link_tracking`：

- `total_onsite_orders`
- `onsite_organic_orders`

如果 `daily_order_link_tracking` 已有当天记录，则更新；如果没有，则创建。

### 4.3 关键词每日明细写回

关键词明细写回 `new_eval_words_daily`。

字段名公式版：

- `new_eval_words_daily.country_asin_date = daily_asins.country_asin_date`
- `new_eval_words_daily.eval_word_id = new_eval_words.id`
- `new_eval_words_daily.date = daily_asins.date`

中文公式版：

- 每个关键词每天一条明细。
- 预计刷单、预测词位、实际刷单、实际词位、复盘、新计划都存在这里。
- 如果当天关键词明细存在则更新，没有则创建。

### 4.4 实际刷单同步逻辑

推新板块会按关键词同步实际刷单。

字段名公式版：

- `evaluation_requirement.asin_country = daily_asins.asin_country`
- `evaluation_requirement.exclusive_evaluation_code = rsg_daily.rsg_id`
- `rsg_daily.date = daily_asins.date`
- `rsg_daily.keywork = new_eval_words.keyword`
- `rsg_daily.id = refund.rsg_daily_id`
- `refund.order_number = order_list.order_number`
- `order_list.status != "Canceled"`
- `new_eval_words_daily.actual_review_qty = COUNT(valid refund rows)`

中文公式版：

- 找到当前 ASIN_国家绑定的测评码。
- 找到同一天、同关键词的 RSG 日数据。
- 找到这些 RSG 日数据关联的返款订单。
- 只有订单号有效且订单未取消的订单才算实际刷单。
- 最后把数量写回对应关键词当天的“实际刷单”。

### 4.5 周汇总覆盖逻辑

周汇总写入 `daily_weekly_summary`。

主键：

- `country_asin_week_range = 国家_ASIN_周起始日_周结束日`

写入字段：

- `push_summary_data`

中文公式版：

- 同一国家、同一 ASIN、同一自然周的日数据合成一条周汇总。
- 已有周汇总则更新，没有则创建。

## 5. 全部静态字段公式清单

| 序号 | 显示名 | 来源字段 | 字段名公式版 | 中文公式版 |
|---:|---|---|---|---|
| 1 | 国家 | `daily.country` | `daily_asins.country` | 从 ASIN 表自动生成并展示国家。 |
| 2 | ASIN | `daily.asin` | `daily_asins.asin` | 从 ASIN 表自动生成并展示 ASIN。 |
| 3 | 站点时间 | `daily.date` | `daily_asins.date` | 每日 8:30 自动生成未来 3 个月日期。 |
| 4 | 销售 | `daily.sale_owner` | `daily_asins.sale_owner` | 从 ASIN 表同步销售负责人。 |
| 5 | 型号 | `daily.model` | `daily_asins.model` | 从 ASIN 表同步型号。 |
| 6 | 活动标注 | `daily.activity_annotation` | `daily_asins.activity_annotation` | 自动匹配 BD/LD 活动日期。 |
| 7 | 推广天数 | `daily.promotion_days` | `daily_asins.promotion_days` | 按首单日期计算推广天数。 |
| 8 | 购物车价格 | `daily.daily_price` | `daily_asins.daily_price` | 自动同步昨日购物车价。 |
| 9 | LP/WP/TP | `daily.list_price` | `daily_asins.list_price` | 自动同步昨日划线价。 |
| 10 | Off 力度 | `daily.off` | `(daily_asins.list_price - daily_asins.daily_price) / daily_asins.list_price` | 划线价减购物车价，再除以划线价。 |
| 11 | 折后售价 | `daily.price_after_discount` | `daily_asins.price_after_discount` | 直接展示折后售价。 |
| 12 | 售卖账号 | `daily.selling_accounts` | `daily_asins.selling_accounts` | 自动同步昨日售卖账号。 |
| 13 | 实际刷单总数 | `daily.rsg_number` | `daily_asins.rsg_number` | 引用订单流量转化里的测评单总数。 |
| 14 | 实际总单量 | `weekly.sales` | `weekly_performance.sales` | 直接展示实际总单量。 |
| 15 | 实际总单量 | `weekly.sales` | `weekly_performance.sales` | 同一个销量字段重复展示，用于不同区域对照。 |
| 16 | 实际自然单 | `weekly.zirandan` | `weekly_performance.zirandan` | 直接展示周表现里的实际自然单。 |
| 17 | 实际广告单 | `weekly.guanggaodan` | `weekly_performance.guanggaodan` | 直接展示广告单量。 |
| 18 | 实际自然单 | `keyword_tracking.actual_natural_order` | `weekly_performance.sales - daily_asins.rsg_number - weekly_performance.guanggaodan` | 总单量扣掉测评单和广告单，得到自然单。 |
| 19 | 实际流量 | `weekly.zongliuliang` | `weekly_performance.zongliuliang` | 直接展示实际流量。 |
| 20 | 实际转化率 | `weekly.zongcvr` | `weekly_performance.zongcvr` | 直接展示实际 CVR。 |
| 21 | 预计刷单总数 | `keyword_tracking.est_review_total` | `SUM(new_eval_words_daily.est_review_qty)` | 当天所有关键词预计刷单量求和。 |
| 22 | 预计流量 | `keyword_tracking.est_traffic` | `daily_keyword_tracking.est_traffic` | 运营维护或已有记录中的预计流量。 |
| 23 | 目标转化率-有备注 | `keyword_tracking.target_cvr` | `daily_keyword_tracking.target_cvr` | 运营维护目标 CVR，可参考前一天实际 CVR 调整。 |
| 24 | 预计自然单 (不含广告单) | `keyword_tracking.est_nat_order` | `daily_keyword_tracking.est_nat_order` | 运营维护预计自然单。 |
| 25 | 单日测评应需量 | `keyword_tracking.daily_eval_demand` | `CEIL(est_traffic * target_cvr - est_nat_order)` | 预计流量乘目标转化率，再减预计自然单，向上取整。 |
| 26 | 计划 - 测评单判断 | `keyword_tracking.plan_eval_judgment` | `COMPARE(daily_eval_demand, est_review_total)` | 对比应该测多少和计划测多少，显示缺单、超单或达标。 |
| 27 | 实际 - 测评结果判断 | `keyword_tracking.actual_eval_judgment` | `COMPARE(daily_asins.rsg_number, est_review_total)` | 对比实际测评和预计测评，显示缺单、超单、无计划或达标。 |
| 28 | 实际测评转化率判断 | `keyword_tracking.actual_cvr_judgment` | `weekly_performance.zongcvr - daily_keyword_tracking.target_cvr` | 实际 CVR 高于目标显示满足，低于目标显示不达标。 |
| 29 | 整体关键词位实际 | `keyword_tracking.actual_keyword_position` | `daily_keyword_tracking.actual_keyword_position` | 展示整体关键词实际位置或截图内容。 |
| 30 | 管理关键词按钮 | `tool.tool_kw_master` | `UI action only` | 页面工具按钮，不是业务数据字段。 |

## 6. 动态关键词字段公式

每个关键词来自 `new_eval_words`，每天明细来自 `new_eval_words_daily`。

| 子字段 | 字段名公式版 | 中文公式版 |
|---|---|---|
| 预计刷单 | `new_eval_words_daily.est_review_qty` | 当前关键词当天计划刷多少单。 |
| 预测词位 | `new_eval_words_daily.est_rank` | 当前关键词预计会到什么词位。 |
| 实际刷单 | `new_eval_words_daily.actual_review_qty` | 按 RSG、返款单、订单状态核算出来的实际刷单数。 |
| 实际词位 | `new_eval_words_daily.actual_rank` | 当前关键词当天实际词位。 |
| 复盘 | `new_eval_words_daily.review_notes` | 运营对当天关键词执行效果的复盘。 |
| 新计划 | `new_eval_words_daily.new_plan` | 后续关键词动作计划。 |

## 7. 关键公式明细

### 7.1 预计刷单总数

字段名公式版：

- `est_review_total = SUM(new_eval_words_daily.est_review_qty)`
- 写回：`daily_keyword_tracking.est_review_total`

中文公式版：

- 把当天所有关键词的“预计刷单”加起来。
- 没有关键词明细时按 0。
- 单个关键词预计刷单为空时按 0。

### 7.2 单日测评应需量

字段名公式版：

- `daily_eval_demand = CEIL(daily_keyword_tracking.est_traffic * daily_keyword_tracking.target_cvr - daily_keyword_tracking.est_nat_order)`
- 写回：`daily_keyword_tracking.daily_eval_demand`

中文公式版：

- 先估算目标转化率下需要多少订单。
- 再减掉预计自然单。
- 得到当天为了达成目标需要补的测评单。
- 结果向上取整。

### 7.3 实际自然单

字段名公式版：

- `actual_natural_order = weekly_performance.sales - daily_asins.rsg_number - weekly_performance.guanggaodan`
- 写回：`daily_keyword_tracking.actual_natural_order`

中文公式版：

- 实际总单量扣掉测评单。
- 再扣掉广告单。
- 剩下就是实际自然单。

### 7.4 计划 - 测评单判断

字段名公式版：

- `diff = daily_eval_demand - est_review_total`
- `daily_eval_demand = 0 => "无计划"`
- `diff > 0 => "计划测评缺-X单"`
- `diff < 0 => "计划测评超+X单"`
- `diff = 0 => "√"`

中文公式版：

- 应该测的比计划多，说明计划缺单。
- 应该测的比计划少，说明计划超单。
- 一样就是计划刚好。
- 应需量为 0 就是无计划。

### 7.5 实际 - 测评结果判断

字段名公式版：

- `diff = actual_review_total - est_review_total`
- `est_review_total = 0 and actual_review_total = 0 => "无计划"`
- `est_review_total = 0 and actual_review_total > 0 => "无计划但测X单"`
- `diff > 0 => "测评超+X单"`
- `diff < 0 => "测评缺-X单"`
- `diff = 0 => "√"`

中文公式版：

- 实际测得比计划多，显示超了多少单。
- 实际测得比计划少，显示缺了多少单。
- 没计划但实际测了，会提醒“无计划但测 X 单”。

### 7.6 实际测评转化率判断

字段名公式版：

- `diff = weekly_performance.zongcvr - daily_keyword_tracking.target_cvr`
- `diff > 0 => "CVR满足+X.XX%"`
- `diff < 0 => "CVR不达标-X.XX%-广告失控？测评不足？"`
- `diff = 0 => "CVR持平"`

中文公式版：

- 实际 CVR 高于目标，表示转化满足。
- 实际 CVR 低于目标，提示转化不达标，并提醒可能是广告失控或测评不足。
- 两者相等时显示持平。

## 8. 小白理解版流程

1. 页面先拿到每天的 ASIN 行。
2. 再把销量、广告单、流量、关键词计划拼进来。
3. 每个关键词每天都可以填预计刷单、预测词位、复盘、新计划。
4. 系统根据 RSG、返款、订单状态核对实际刷单。
5. 系统用预计流量、目标 CVR、预计自然单算出当天应该测多少单。
6. 系统对比计划和实际，告诉运营缺单、超单还是达标。
7. 同一自然周的数据会形成周汇总，方便看一周推新效果。
