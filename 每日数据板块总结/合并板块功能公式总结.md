# 合并板块功能公式总结

## 1. 板块一句话说明

合并板块把每日 ASIN 主表、周表现、目标管理、利润、订单链接跟踪、商品配置、SQP 关键词自然位、竞对跟踪等数据合到同一张表里，方便运营人员按「国家 + ASIN + 日期」查看销量、流量、广告、测评、利润、目标达成和链接问题。

这个板块不是单一数据表，而是一个“综合看板”。主线数据来自 `daily_asins`，其他表通过同一个日期主键或 ASIN 主键拼进来。

## 2. 主要功能

### 2.1 基础筛选与定位

- 支持从 URL 参数读取 `model`、`country`、`asin`、`sale_owner`。
- URL 没有参数时，会从页面缓存读取上一次使用的型号、国家、ASIN、销售。
- 当前用户为销售层级时，只读取 `daily_asins.sale_owner = 当前用户名` 的记录。
- 默认按日期升序显示。
- 支持分页，默认每页 20 条，可选 10、20、50、100。

### 2.2 数据合并

一行代表一个 `国家 + ASIN + 日期`。

核心主键：

- 日维度主键：`country_asin_date`
- ASIN 国家主键：`asin_country = ASIN_国家`
- 国家 ASIN 主键：`country_asin = 国家_ASIN`

合并规则：

- `daily_asins.country_asin_date` 对应 `weekly_performance.country_asin_week`
- `daily_asins.country_asin_date` 对应 `target_management.country_asin_date`
- `daily_asins.country_asin_date` 对应 `daily_profit.country_asin_date`
- `daily_asins.country_asin_date` 对应 `daily_order_link_tracking.country_asin_date`
- `daily_asins.asin_country` 对应 `product_config.asin_country`
- `国家_ASIN` 对应 `target_default.country_asin`
- `国家_ASIN` 对应 `sqp_keywords.country_asin`
- `国家_ASIN` 对应 `order_link_competitor_asins.country_asin`

### 2.3 公式同步

合并板块会计算并写回多类公式：

- 日表公式：写回 `daily_asins`
- 目标公式：写回或创建 `target_management`
- 订单链接公式：写回或创建 `daily_order_link_tracking`
- 利润公式：写回或创建 `daily_profit`
- 周汇总：写回或创建 `daily_weekly_summary`

公式写回不是无脑覆盖。脚本会先比较旧值和新值，只有不同才排队写回。

### 2.4 动态列

合并板块会根据当前 ASIN 自动生成动态列：

- SQP 关键词自然位列：来自 `sqp_keywords` 和 `sqp_keyword_daily_positions`
- 竞对 ASIN 列：来自 `order_link_competitor_asins` 和 `order_link_competitor_asins_daily`

动态列不是固定字段。添加关键词或竞对后，页面会根据记录自动生成列。

### 2.5 周汇总行

合并板块会按自然周生成周汇总行。

自然周规则：

- 周起始：周日
- 周结束：周六
- 自定义日期也会扩展到完整自然周

例如选择周三到周五，实际周汇总会按该周周日到周六计算。

### 2.6 目标值管理

合并板块提供“管理目标值”功能，用来维护当前国家 ASIN 的默认目标。

目标值保存到：

- `target_default`

可维护字段：

- 目标广告 CVR：`target_ad_cvr`
- 目标 CPA：`target_cpa`
- 目标 CPU：`ideal_cpu_by_margin`
- 目标利润率：`target_profit_margin`
- 目标广告费率：`target_ad_spend_rate`

字段名公式版：

- `target_default.country_asin = 当前国家_ASIN`
- `target_default.country = 当前国家`
- `target_default.asin = 当前 ASIN`
- `target_ad_cvr = 输入百分比 / 100`
- `target_profit_margin = 输入百分比 / 100`
- `target_ad_spend_rate = 输入百分比 / 100`
- `if target_default exists => update`
- `else => create`

中文公式版：

- 运营在弹窗里维护当前 ASIN 的目标广告 CVR、CPA、CPU、利润率和广告费率。
- 百分比类字段页面上输入 5%，数据库里按 0.05 保存。
- 保存后刷新合并板块，目标判断公式会按新目标重新计算。

### 2.7 Coupon 预估比例管理

合并板块提供“管理 Coupon 预估比例”功能，用来维护当前 ASIN_国家的 Coupon 订单比例。

写回表：

- `product_config`

写回字段：

- `product_config.coupon_order_ratio_estimated`

字段名公式版：

- `product_config.asin_country = 当前 ASIN_国家`
- `coupon_order_ratio_estimated = 输入百分比 / 100`
- `coupon_total_cost = price_after_discount * coupon_commission_rate * (sales - rsg_number) * coupon_order_ratio_estimated`

中文公式版：

- 运营维护“有多少真实订单会产生 Coupon 费用”的预估比例。
- 保存后会立刻影响 Coupon 总费用。
- Coupon 总费用变化后，利润、利润率、广告费率等关联公式也会跟着重新计算。

### 2.8 SQP 词自然位管理

合并板块提供“追踪 SQP 词自然位”功能，用来维护当前 ASIN 下要追踪的 SQP 关键词或词根，并记录每天的自然排名。

配置表：

- 关键词：`sqp_keywords`
- 词根：`sqp_roots`

每日自然位表：

- `sqp_keyword_daily_positions`

字段名公式版：

- `sqp_keywords.country_asin = 当前国家_ASIN`
- `sqp_roots.country_asin = 当前国家_ASIN`
- `sqp_keyword_daily_positions.sqp_keyword_id = sqp_keywords.id`
- `sqp_keyword_daily_positions.country_asin_date = daily_asins.country_asin_date`
- `sqp_keyword_daily_positions.actual_rank = 页面填写的自然位`

中文公式版：

- 运营先维护需要追踪的 SQP 词。
- 每个词会在合并表里生成一个动态自然位列。
- 每天的实际自然位保存在 `sqp_keyword_daily_positions.actual_rank`。
- 如果当天已有自然位记录，就更新；如果没有，就创建。

### 2.9 竞对 ASIN 管理

合并板块提供“管理竞对 ASIN”功能，用来维护当前 ASIN 对应的竞品链接，并记录竞对每天的排名和操作分析。

配置表：

- `order_link_competitor_asins`

每日记录表：

- `order_link_competitor_asins_daily`

字段名公式版：

- `order_link_competitor_asins.country_asin = 当前国家_ASIN`
- `order_link_competitor_asins.role = 竞对1/竞对2/...`
- `order_link_competitor_asins.competitor_asin = 输入的竞对 ASIN`
- `order_link_competitor_asins.notes = 列头备注`
- `order_link_competitor_asins_daily.competitor_id = order_link_competitor_asins.id`
- `order_link_competitor_asins_daily.country_asin_date = daily_asins.country_asin_date`
- `order_link_competitor_asins_daily.rank = 当天竞对排名`
- `order_link_competitor_asins_daily.notes = 当天操作分析`

中文公式版：

- 运营给当前 ASIN 维护一个或多个竞对 ASIN。
- 每个竞对会生成动态列，包含排名和操作分析。
- 每天的竞对排名和操作分析保存到 `order_link_competitor_asins_daily`。
- 删除某个竞对时，会同步删除这个竞对已经生成的每日记录，避免页面残留旧列旧数据。

### 2.10 视图、列设置、推送配置和高亮

合并板块支持保存页面展示配置。

配置内容包括：

- 完整列视图。
- 核心列视图。
- 自定义视图。
- 列显示、隐藏、宽度、固定列。
- 字段分组顺序。
- 每页显示数量。
- 列头颜色。
- 重要指标整列背景。
- 高亮当前行列。
- 管理员推送完整列和核心列配置给其他用户。

这些配置主要写入：

- `users.setting`

字段名公式版：

- `users.setting[BLOCK_NAME_SETTING_KEY] = "合并板块"`
- `users.setting[COLUMN_VIEW_SETTING_KEY] = 当前用户视图配置`
- `users.setting[DEFAULT_COLUMN_VIEWS_KEY] = 完整列/核心列默认配置`
- `users.setting[COLUMN_GROUP_ORDER_KEY] = 字段分组顺序`
- `users.setting[COLUMN_PAGE_SIZE_KEY] = pageSize`

中文公式版：

- 列设置只影响页面怎么展示，不改变业务数据。
- 列头颜色保存为 `headerColor`，只改变列头底色。
- 重要指标整列背景保存为 `bodyColor`，固定颜色为 `#BADDB1`。
- 重要指标背景只改变表体单元格，不改变列头。
- 选中单元格、高亮当前行列、周汇总行的临时底色优先于重要指标背景。
- 管理员保存默认视图后，可以推送给其他用户。
- 推送配置会同步目标用户的完整列和核心列默认视图。
- 推送配置还会把列头颜色和重要指标背景同步到目标用户已有自定义视图。
- 目标用户自定义视图里的显示隐藏、固定列、列宽、编辑开关、板块顺序、每页数量会尽量保留。
- 推送配置不会改销量、利润、目标、竞对等业务表。

## 3. 数据什么时候抓

### 3.1 页面打开时即时抓取

页面加载、切换筛选、翻页、刷新、切回页面可见时，会即时读取相关数据表。

即时读取的主要表：

- `daily_asins:list`
- `weekly_performance:list`
- `target_management:list`
- `target_default:list`
- `daily_profit:list`
- `daily_order_link_tracking:list`
- `product_config:list`
- `sqp_keywords:list`
- `sqp_keyword_daily_positions:list`
- `order_link_competitor_asins:list`
- `order_link_competitor_asins_daily:list`
- `daily_weekly_summary:list`

操作管理功能会在点击按钮或保存时额外读取或写入：

- `target_default:list/create/update`
- `product_config:list/update`
- `sqp_keywords:list/create/update/destroy`
- `sqp_roots:list/create/update/destroy`
- `sqp_keyword_daily_positions:list/create/update`
- `order_link_competitor_asins:list/create/update/destroy`
- `order_link_competitor_asins_daily:list/create/update/destroy`
- `users:get`
- `users:list`
- `users:update`

### 3.2 自动任务抓取时间

脚本中记录的自动来源如下：

- `daily_asins.date`：每日 8:30 自动生成从当天起未来 3 个月的数据。
- `daily_asins.activity_annotation`：每日 8:30 自动匹配 BD/LD 活动日期。
- 非 US/CA 站点的购物车价、LP/WP/TP、星级、Review 数、售卖账号、推广天数：每日 8:30 自动同步昨日数据。
- US/CA 站点的购物车价、LP/WP/TP、星级、Review 数、售卖账号、推广天数：每日 13:00 自动同步昨日数据。
- `target_management.sales_mom_rate`：每日 8:30 自动对比当天和前一天实际总单量。

### 3.3 默认抓取时间范围

页面默认日期筛选为「近7天及以后」。

字段名公式版：

- `start_date = today - 6 days`
- `end_date = null`
- `daily_asins.date >= start_date`
- 日期范围再扩展到自然周：`week_start = selected_start 所在周周日`，`week_end = selected_end 所在周周六`

中文公式版：

- 默认从页面打开当天往前数 6 天开始，一直取到未来已有数据。
- 为了能生成周汇总，实际读取周汇总相关数据时，会把日期扩到完整自然周。

### 3.4 可选日期范围

- 全部日期：不加日期条件。
- 今天：`date = 今天`
- 昨天：`date = 昨天`
- 近 7 天：`今天 - 6 天 <= date <= 今天`
- 近 30 天：`今天 - 29 天 <= date <= 今天`
- 近 90 天：`今天 - 89 天 <= date <= 今天`
- 本月：`本月 1 日 <= date <= 今天`
- 上月：`上月 1 日 <= date <= 上月最后一天`
- 自定义：用户选定开始和结束日期，再扩展到自然周。

## 4. 数据覆盖与写回逻辑

### 4.1 只展示不写回

大部分周表现字段、基础字段、商品配置字段只在页面展示，不会因为打开页面而被覆盖。

例如：

- `weekly_performance.sales`
- `weekly_performance.guanggaohuafei`
- `weekly_performance.ranking`
- `product_config.refund_rate_new`
- `target_default.target_cpa`

### 4.2 有差异才写回

公式类字段会先计算新值，再和数据库已有值比较。

字段名公式版：

- `if String(old_value ?? '') !== String(new_value ?? '') then write`
- `if old and new 都为空 then skip`

中文公式版：

- 如果旧值和新值一样，不写库。
- 如果旧值和新值不同，才写回对应表。

### 4.3 只更新的表

`daily_asins` 只更新已有记录，不在合并板块里创建新日表记录。

写回字段：

- `rsg_number`
- `off`
- `promo_day`
- `lp_duration_days`
- `promo_days_40d`
- `promo_days_90d`
- `target_gap`

### 4.4 可创建或更新的表

以下表有记录时更新，没有记录时创建：

- `target_management`
- `daily_order_link_tracking`
- `daily_profit`
- `daily_weekly_summary`

### 4.5 测评单同步覆盖逻辑

合并板块会用返款和订单数据重新计算 `daily_asins.rsg_number`。

字段名公式版：

- `evaluation_requirement.asin_country = daily_asins.asin_country`
- `evaluation_requirement.exclusive_evaluation_code = rsg_daily.rsg_id`
- `rsg_daily.date = daily_asins.date`
- `rsg_daily.id = refund.rsg_daily_id`
- `refund.order_number = order_list.order_number`
- `order_list.status != "Canceled"`
- `rsg_number = COUNT(valid refund rows)`

中文公式版：

- 找到当前 ASIN_国家对应的测评需求。
- 用专属测评码找到同一天的 RSG 日数据。
- 再找这些 RSG 日数据关联的返款订单。
- 只有订单号存在、且订单状态不是取消的订单才计入测评单。
- 统计出来的数量写回 `daily_asins.rsg_number`。

### 4.6 周汇总覆盖逻辑

周汇总写入 `daily_weekly_summary`。

主键：

- `country_asin_week_range = 国家_ASIN_周起始日_周结束日`

字段名公式版：

- `summary_group = same country + same asin + natural_week(date)`
- `daily_weekly_summary.merge_summary_data = summary(rows in summary_group)`

中文公式版：

- 把同一个国家、同一个 ASIN、同一个自然周的日行放在一起。
- 对订单、广告、利润等指标求和或求均值。
- 对目标达成、广告预算等字段按周汇总逻辑重新计算。
- 如果该周已有汇总记录，则更新；如果没有，则创建。

### 4.7 目标默认值覆盖逻辑

目标默认值写入 `target_default`。

字段名公式版：

- `target_default.country_asin = 当前国家_ASIN`
- `if target_default.country_asin exists => target_default:update`
- `else => target_default:create`
- `target_ad_cvr = 输入百分比 / 100`
- `target_cpa = 输入金额`
- `ideal_cpu_by_margin = 输入金额`
- `target_profit_margin = 输入百分比 / 100`
- `target_ad_spend_rate = 输入百分比 / 100`

中文公式版：

- 当前 ASIN 已经有目标默认值时，保存会更新旧记录。
- 当前 ASIN 没有目标默认值时，保存会新建一条。
- 保存后不会直接改历史销量，而是让目标判断公式按新的目标重新计算。

### 4.8 Coupon 预估比例覆盖逻辑

Coupon 预估比例写入 `product_config`。

字段名公式版：

- `product_config.asin_country = 当前 ASIN_国家`
- `product_config.coupon_order_ratio_estimated = 输入百分比 / 100`
- `coupon_total_cost = price_after_discount * coupon_commission_rate * (sales - rsg_number) * coupon_order_ratio_estimated`

中文公式版：

- 当前 ASIN_国家必须已经有产品配置记录，才能保存 Coupon 预估比例。
- 保存后只更新 `coupon_order_ratio_estimated`。
- Coupon 总费用、纯利润、利润率等公式会根据新比例重新计算。

### 4.9 SQP 词配置和自然位覆盖逻辑

SQP 词配置写入 `sqp_keywords` 或 `sqp_roots`，每日自然位写入 `sqp_keyword_daily_positions`。

字段名公式版：

- `keyword/root.country_asin = 当前国家_ASIN`
- `keyword/root.country = 当前国家`
- `keyword/root.asin = 当前 ASIN`
- `daily_position.sqp_keyword_id = keyword/root.id`
- `daily_position.country_asin_date = daily_asins.country_asin_date`
- `if daily_position exists => update actual_rank`
- `else => create actual_rank`

中文公式版：

- 新增 SQP 词后，页面会为这个词生成自然位动态列。
- 运营填写某天自然位时，如果当天已有记录，就更新。
- 如果当天没有记录，就新建一条。
- 删除 SQP 词时，词配置会被删除；页面重新加载后不再显示该词动态列。

### 4.10 竞对 ASIN 覆盖和清理逻辑

竞对配置写入 `order_link_competitor_asins`，每日竞对数据写入 `order_link_competitor_asins_daily`。

字段名公式版：

- `competitor.country_asin = 当前国家_ASIN`
- `competitor.role = 竞对序号`
- `competitor.competitor_asin = 输入竞对 ASIN`
- `competitor.notes = 列头备注`
- `daily.competitor_id = competitor.id`
- `daily.country_asin_date = daily_asins.country_asin_date`
- `if daily exists => update rank/notes`
- `else => create rank/notes`
- `delete competitor => delete all daily rows where competitor_id = competitor.id`

中文公式版：

- 新增竞对后，页面会生成该竞对的动态列。
- 竞对 ASIN 或列头备注修改后，写回竞对配置表。
- 每天的竞对排名和操作分析保存在每日竞对表。
- 删除竞对时，系统会先删除该竞对所有每日记录，再删除竞对配置，避免旧数据残留。

### 4.11 视图配置覆盖逻辑

页面视图配置写入 `users.setting`。

字段名公式版：

- `users.setting[BLOCK_NAME_SETTING_KEY] = "合并板块"`
- `users.setting[COLUMN_VIEW_SETTING_KEY] = personal_column_views`
- `users.setting[DEFAULT_COLUMN_VIEWS_KEY] = default_column_views`
- `users.setting[COLUMN_GROUP_ORDER_KEY] = column_group_order`
- `users.setting[COLUMN_PAGE_SIZE_KEY] = page_size`

中文公式版：

- 用户保存个人视图时，只更新当前用户的设置。
- 管理员保存默认视图时，会更新默认完整列或核心列配置。
- 管理员推送配置时，会把完整列和核心列配置同步给选中的用户。
- 列头颜色保存为 `headerColor`，重要指标背景保存为 `bodyColor`。
- 推送默认配置时，`headerColor` 和 `bodyColor` 会同步到目标用户已有自定义视图。
- 目标用户自定义视图里的列显示、固定列、列宽、编辑开关、板块顺序和分页不跟随样式推送强制覆盖。
- 这些配置只影响列显示、列宽、列顺序、颜色、重要指标背景和分页，不影响业务表数据。

### 4.12 高亮、列头颜色和重要指标背景覆盖逻辑

高亮行列、列头颜色和重要指标背景属于展示设置。

字段名公式版：

- `header_color_map = column payload headerColor`
- `body_color_map = column payload bodyColor`

## 6. 动态字段公式

### 6.1 SQP 关键词自然位

动态列来源：

- 关键词配置：`sqp_keywords`
- 每日词位：`sqp_keyword_daily_positions`

字段名公式版：

- `column = kw_actual_{sqp_keywords.id}`
- `value = sqp_keyword_daily_positions.actual_rank`
- `match = sqp_keyword_daily_positions.sqp_keyword_id = sqp_keywords.id AND sqp_keyword_daily_positions.date = daily_asins.date`

中文公式版：

- 当前 ASIN 下每维护一个 SQP 关键词，页面就生成一列“词X:关键词 自然位”。
- 每天显示该关键词当天的实际自然排名。
- 该列可编辑，保存后写回 `sqp_keyword_daily_positions.actual_rank`。
- 如果当天没有该关键词的词位记录，保存时会创建。

### 6.2 竞对 ASIN 字段

动态列来源：

- 竞对配置：`order_link_competitor_asins`
- 竞对每日记录：`order_link_competitor_asins_daily`

每个竞对生成两个子字段：

- 排名：`order_link_competitor_asins_daily.rank`
- 操作分析：`order_link_competitor_asins_daily.notes`

字段名公式版：

- `column = competitor_dynamic_{competitor_id}_rank`
- `rank = order_link_competitor_asins_daily.rank`
- `notes = order_link_competitor_asins_daily.notes`
- `match = competitor_id + country_asin_date`

中文公式版：

- 当前 ASIN 下每维护一个竞对 ASIN，页面就生成一组竞对列。
- “排名”记录竞对当天排名。
- “操作分析”记录竞对当天动作或分析。
- 保存时按竞对 ID 和当前日期写回每日竞对表。

## 8. 小白理解版流程

1. 页面先从每日 ASIN 主表拿到日期行。
2. 再按同一个日期主键，把销量、目标、利润、订单链接等数据拼到同一行。
3. 再用测评需求、RSG、返款、订单表重新核对测评单数量。
4. 再用定价试算表匹配普通售价和测评售价，算利润、费用、成交额。
5. 再用目标默认值和实际广告表现，判断目标是否达标。
6. 如果运营维护了目标默认值，目标判断会按新的默认值重新算。
7. 如果运营维护了 Coupon 预估比例，Coupon 费用和利润会按新比例重新算。
8. 如果运营维护了 SQP 词，页面会多出自然位动态列。
9. 如果运营维护了竞对 ASIN，页面会多出竞对排名和操作分析动态列。
10. 再把同一自然周的数据合成周汇总行。
11. 最后只把发生变化的公式字段写回数据库，没变化的不动。
12. 列设置、视图、列头颜色、重要指标背景、页大小和推送配置只改变页面展示，不改变业务数据。
